Assignment Concept Questions

1.Why is textContent safer than innerHTML?

Answer: textContent treats all input strictly as plain text, rendering code characters literally. innerHTML parses text as HTML markup. If you use innerHTML with unvalidated user input, it leaves the page open to Cross-Site scripting (XSS) attacks, where malicious actors can inject and execute arbitrary JavaScript scripts inside the browser.

2. How are HTML elements connected to JavaScript variables using DOM selection?

Answer:Connection is achieved using DOM selection methods provided by the global documentobject, such as document.getElementById() or document.querySelector(). These methods search the DOM tree for nodes matching the specified ID, class, or selector and return a reference to that element, which is then assigned to a JavaScript variable.

3. What would happen to the text box with id="text" if I added this line to the code above it: text.textContent = "Hello!"

Answer: The browser would immediately overwrite whatever content was previously nested inside the element with the ID text and replace it with the plain string "Hello!". If the variable text had not been declared beforehand, modern browsers might attempt to resolve the ID globally, but it is best practice to select the element explicitly first.
